# Product — Admin

루트 [`../PRODUCT.md`](../PRODUCT.md) 를 상속한다. 이 표면은 **product** 레지스터, 데스크톱 우선.

## Register

product

## Platform

web

## Users

운영 관리자. 데스크톱에서 **기관·의료진 승인, 사용자 상태 관리, 리뷰 관리, 정산 리포트**를 처리한다.
하루에 여러 번 목록을 훑고 소수의 항목에 대해 결정(승인/반려/정지)을 내린다.
숙련 사용자이므로 안내보다 **밀도와 속도**가 중요하다.

## Product Purpose

플랫폼의 신뢰를 유지하는 **게이트키핑 도구**. 미검증 의료진이 진료하지 못하게 막는 것이 존재 이유다.
성공은 "결정까지 걸린 클릭 수와 시간"으로 측정된다.

## Brand Personality

도구는 과업 뒤로 사라져야 한다. 개성보다 **일관성**이 미덕이다.
Linear·Stripe 대시보드처럼 익숙하고 예측 가능한 어휘를 쓴다.

## Anti-references

- 관리자 화면에 브랜드 히어로·마케팅 카피
- 콘텐츠 자리의 스피너 (스켈레톤으로 교체 완료 — 되돌리지 말 것)
- "없습니다"로 끝나는 빈 상태
- 사용자에게 의미 없는 내부 정보 노출 (예: `localhost:8080` — 제거 완료)
- 표준 어포던스 재발명(커스텀 스크롤바, 기이한 폼 컨트롤), 모달 남용

## Design Principles

1. **결정을 앞으로.** 상태 배지와 승인/반려 액션이 한눈에. 상세는 그 다음.
2. **밀도는 미덕.** 목록은 조밀하게. 여백으로 정보를 밀어내지 않는다.
3. **모든 상태를 설계한다.** 기본·호버·포커스·비활성·로딩·빈 상태·오류 전부. 반쯤 만든 컴포넌트는 없다.
4. **화면 간 어휘 통일.** 같은 동작은 같은 버튼 모양·같은 배지 톤.
5. **되돌릴 수 없는 일은 확인받는다.** 정지·삭제는 결과를 명시하고 확인을 거친다.

## Accessibility & Inclusion

WCAG 2.1 AA. 상태는 색상 단독 금지(배지 텍스트 동반).
로딩은 `aria-busy`/`aria-live`, 스켈레톤은 `motion-safe`.
키보드만으로 목록 훑기와 승인/반려가 가능해야 한다.
